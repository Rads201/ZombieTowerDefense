About |

This game was created as a Capstone project for CS467 at Oregon State University. The project description was to create an HTML5 web-based tower defense game that adhered to the following parameters:

  • Game must be single player.
  • Game must allow at least different types of individual towers to be constructed that serve some purpose on a 2D board of level.
  • There must be at least 3 different boards or levels.
  • Enemy units will enter the play field and attempt to achieve some objective. Towers will attempt to stop the enemy units from completing this objective in order to win.
  • There must be at least 10 waves of enemies that enter the board or level.
  • There must be at least 3 different types of enemy units.
  • Each defeated enemy should provide some sort of benefit to the player.
  • If the game has units attacking or otherwise affecting something, appropriate animations showing projectiles must be shown to indicate what's happening.
  • An individual game screen, or board must be winnable, with reasonable to serious effort/skill, within 10 minutes.
  • A new player should not be able to beat the game.
  • Game must be played using the Google Chrome browser.
  • Game must be hosted somewhere for testing purposes.
  • Usage of game libraries, frameworks, and toolkits is allowed as well as general purpose libraries.
  • Do not use Adobe Flash.
  • Do not use Javascript alert() popups.
  • Data about individual enemies and towers must be loaded from separate files on the web server (one unit per file).
